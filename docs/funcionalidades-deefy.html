Análise atual

Pontos de atenção

Itens que funcionam, mas merecem cuidado porque impactam fluidez, estabilidade ou evolução.

Performance

Configurações carregam perfil via API a cada montagem

/configuration, /custom-profile e /edit-profile buscam /users/me individualmente. A interface poderia usar cache local primeiro e atualizar em background para parecer instantânea.

Player

Player é desmontado em rotas fora da lista permitida

O áudio vive dentro do componente visual do player. Quando a rota não renderiza MusicPlayer, o elemento <audio> some e a música pode parar ou reiniciar ao navegar.

Busca

Busca ampla dispara muitas chamadas paralelas

A busca global pode consultar músicas, artistas, playlists, gênero e álbum. Funciona, mas pode gerar delays perceptíveis sem cache ou cancelamento de requisições antigas.

Favoritos

Estado de favoritos é consultado em várias montagens

Listas de músicas carregam favoritos para marcar os corações. O comportamento é correto, mas pode ficar mais rápido com cache centralizado por usuário.

Modelo de dados

Álbum ainda é uma compatibilidade visual

O fluxo principal em produção não tem tabela física de álbum consolidada. Hoje a UI usa gênero e playlists globais para preservar a ideia visual de lançamento/álbum.
