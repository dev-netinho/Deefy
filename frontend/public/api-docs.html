Deefy API publica

Contrato oficial da staging

Esta documentacao descreve somente a API oficial disponivel em producao/staging para o time de frontend testar localmente. Endpoints experimentais da sandbox nao fazem parte deste contrato.

Base URL: https://deefy.olua.me/api/v1 Auth: JWT Bearer Frontend local: liberado via CORS

Como testar no frontend local

Crie/ajuste o arquivo frontend/.env apontando para a API publica:

VITE_API_URL=https://deefy.olua.me
VITE_STORAGE_TOKEN_KEY=@deefy-token

Depois rode o projeto normalmente:

cd frontend
npm install
npm run dev

Origens liberadas para navegador: http://localhost:5173, http://localhost:5174, http://127.0.0.1:5173 e https://deefy.olua.me.

Fluxo de autenticacao

POST /auth/register

Cria uma nova conta.

{
  "nome": "Maria Silva",
  "email": "maria@example.com",
  "senha": "Senha@123"
}

Resposta esperada: 201 Created

{
  "id": 6,
  "nome": "Maria Silva",
  "email": "maria@example.com",
  "createdAt": "2026-05-17T01:35:28.593343498"
}
POST /auth/login

Autentica com email e senha e devolve um JWT.

{
  "email": "maria@example.com",
  "senha": "Senha@123"
}

Resposta esperada: 200 OK

{
  "token": "eyJhbGciOiJIUzI1NiJ9..."
}

Como enviar o token

Todos os endpoints fora de /auth/** exigem header de autorizacao:

Authorization: Bearer <token>

Exemplo com Axios:

import axios from "axios";

export const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
});

api.interceptors.request.use((config) => {
  const token = localStorage.getItem("@deefy-token");
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

Usuarios

GET /users?page=0&size=5&sort=id,asc

Lista usuarios de forma paginada. Requer token.

GET /users/{id}

Busca um usuario por id. Requer token.

PUT /users/{id}

Atualiza nome e/ou senha. Requer token.

{
  "nome": "Novo Nome",
  "password": "NovaSenha@123"
}
DELETE /users/{id}

Remove um usuario. Requer token. Resposta esperada: 204 No Content.

Exemplos com cURL

curl -X POST https://deefy.olua.me/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"nome":"Maria Silva","email":"maria@example.com","senha":"Senha@123"}'
curl -X POST https://deefy.olua.me/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"maria@example.com","senha":"Senha@123"}'
curl https://deefy.olua.me/api/v1/users \
  -H "Authorization: Bearer <token>"

O que ainda nao e API oficial

Catalogo musical

/artists, /albums, /musics e upload MinIO ainda nao estao expostos na API oficial da staging.

Playlists e historico

Existem classes internas parciais, mas ainda nao existe controller oficial publicado para frontend consumir.

Status HTTP comuns

  • 200: requisicao concluida com sucesso.
  • 201: recurso criado com sucesso.
  • 204: recurso removido sem corpo de resposta.
  • 400/422: payload invalido ou regra de validacao falhou.
  • 403: token ausente, invalido ou endpoint nao liberado para uso.
  • 500: erro inesperado no backend.
